# PRD: CafeMinsu (카페민수)

> 본 PRD의 **화면 상세 디자인 단일 진실은 `docs/screens/`**(21개 PNG)이며, 텍스트 스펙은 `docs/SCREENS.md`다.
> 디자인과 본 문서가 충돌하면 **디자인이 우선**한다. (이 개정은 `docs/screens` 디자인에 맞춰 정렬됨.)

## 목표
동네 카페 "민수"의 단골이 **로그인 → 매장 선택 → 메뉴 주문·결제, 스탬프·쿠폰 적립, 기프티콘 선물,
음성 AI 주문**을 한 앱에서 빠르고 따뜻하게 처리하도록 한다. (네이티브 Android, 모바일 우선)

## 사용자
- **주 사용자**: 카페 민수를 반복 방문하는 단골 / 인근 직장인·주민 (모바일로 미리 주문하고 픽업)
- 1차 페르소나: "줄 서기 싫고, 적립은 챙기고 싶은 단골"
- (점주/매장 운영자는 MVP 대상 아님)

## 핵심 기능 (MVP)
1. **로그인 / 온보딩** — 스플래시 → **카카오 로그인**으로 진입. 미인증 시 로그인 게이트.
2. **매장 선택** — 주문 시작 시 지도·근처 매장 목록·검색으로 매장 선택 → 매장 상세(영업시간·편의시설) 확인 후 주문 시작.
3. **메뉴 탐색 + 주문 + 결제** — 카테고리별 메뉴 → 옵션(온도·사이즈·샷) 선택 → 장바구니(매장식사/포장·요청사항)
   → 결제(신용카드/간편결제/쿠폰) → 주문 성공/실패. 결제 성공 시 스탬프 자동 적립.
4. **스탬프 & 쿠폰 & 기프티콘 선물** — 결제 시 스탬프 적립, 쿠폰(무료음료·금액형) 보관/사용, 금액형 기프티콘을
   **카카오톡/문자로 선물**.
5. **음성 AI 주문** — 음성 발화 → STT → 주문 파싱(메뉴·옵션·수량) → 인식 결과·신뢰도·예상 금액 확인 → 장바구니 반영.
6. **알림 / 주문 추적** — 주문 수락·준비·완료 알림, 주문내역에서 진행중 주문 라이브 트래킹 + 지난 주문 재주문.

## MVP 제외 사항
- **키오스크 모드**(매장 단말용) — 추후 별도 트랙
- 배달 / 외부 배달앱 연동, 점주 **고급** 운영(자동 정산·멀티매장·직원 권한 관리)
  - 단, 점주 **기본 운영 화면**(로그인·대시보드·실시간 주문·메뉴·매출)은 본 트랙에 포함 → 아래 "점주(매장) 화면 인벤토리"
- 멤버십 등급 캠페인·추천인 (등급 표시는 노출하되 정책은 후속), 다국어(한국어 단일)

### 점주(매장) 화면 인벤토리
점주(매장 관리자)용 화면. 로그인 화면 하단 **"점주 로그인 →"** 링크로 진입하며, 고객(카카오)과 분리된 **아이디/비밀번호** 인증을 쓴다. 하단 탭은 **대시보드 · 주문 · 메뉴 · 매출 (4개)**.

| 코드           | 화면              | 핵심 내용                                                       | 디자인 파일                          |
| -------------- | ----------------- | -------------------------------------------------------------- | ------------------------------------ |
| `OWNER_LOGIN`  | 점주 로그인       | 아이디/비밀번호 폼, 로그인, 고객 로그인 복귀 링크              | `점주 - 00 (로그인).png`             |
| `OWNER_HOME`   | 점주 대시보드     | 영업중 토글, 오늘 매출·주문·신규 대기, 처리할 주문, 빠른 액션   | `점주 - 01 (대시보드).png`           |
| `OWNER_ORDERS` | 실시간 주문 관리  | 신규·준비중·준비완료 필터, 주문 카드, 상태별 처리              | `점주 - 02 (주문 관리).png`          |

> 주문 상태 변경(접수→준비완료→픽업완료)·품절 처리 등 운영 액션도 **낙관적 UI 금지**(서버 확정 후 반영).

## 사용자 플로우 (핵심)
1. **온보딩**: `SPLASH` → (미인증) `LOGIN`(카카오) → `HOME`
2. **표준 주문**: `HOME`/주문 탭 → `STORE` → `STORE_DETAIL` → `MENU` → `MENU_DETAIL`(담기) → `CART`(매장식사/포장)
   → `PAY` → `ORDER_OK`(스탬프 적립) — 실패 시 `ORDER_FAIL`(다시 시도)
3. **음성 주문**: `MENU` → `VOICE`(발화·인식 결과/신뢰도 확인) → 장바구니 반영 → `CART` → …
4. **기프티콘 선물**: `MY` → `GIFT`(금액 선택 → 카카오톡/SMS 수신자 → 선물)
5. **적립/쿠폰**: `MY` → `COUPON`(스탬프 진행도 + 쿠폰 목록/사용)
6. **주문 추적**: `NOTI` 또는 `MY` → `HISTORY`(진행중 주문 단계: 접수→수락→준비중→완료) + 재주문
7. **점주 운영**: `LOGIN`(점주 로그인 링크) → `OWNER_LOGIN`(아이디/비밀번호) → `OWNER_HOME` → 실시간
   `OWNER_ORDERS`(접수→준비완료→픽업완료) / `OWNER_MENU`(품절 토글) / `OWNER_SALES`(매출·정산 확인)
8. **주요 예외**: 품절·영업마감 차단/안내, 결제 실패·중복 방지(멱등키), 세션 만료 재로그인, 네트워크 오류(`ST_NETERR`),
   권한 거부(마이크) 대체 경로. (처리 원칙 `ARCHITECTURE.md` 에러/엣지 절, 보안 `SECURITY.md`)

## 디자인
- 방향: **웜 크림 + 코랄 + 다크 네이비** 에디토리얼 (차분하고 손맛 있는 카페 톤)
- **화면 상세 디자인 단일 진실: `docs/screens/`(PNG) + `docs/SCREENS.md`(텍스트 스펙).** 토큰·컴포넌트는 `docs/DESIGN_SYSTEM.md`.
- 화면 적용·안티슬롭 원칙: `docs/UI_GUIDE.md`

## 가정 / 결정 (구현 기준)
- **로그인**: 카카오 로그인 채택(실연동). Kakao Native App Key 필요 → 키 부재 시 Mock 로그인 폴백, 실동작은 키 수급 후.
- **점주 로그인**: 고객(카카오)과 분리된 **아이디/비밀번호** 인증. 백엔드 전까지 Mock 로그인. 점주 세션도 EncryptedDataStore 저장 + 역할(`CUSTOMER`/`OWNER`) 구분, 점주 전용 라우트/하단 탭으로 분리.
- **매장 지도**: 지도 SDK 실연동(Maps API 키 필요) → 키 부재 시 스타일드 플레이스홀더 지도 폴백.
- **카카오톡 선물**: Kakao 공유/메시지 실연동(키 필요) → 폴백 시 Mock 전송.
- **결제(PG)**: **Mock**(PG 미연동, 성공/실패 분기) — `ADR-006`/`ADR-010`. 카드 PAN/CVC 미저장·미로깅, 멱등키.
- **음성 AI**: 온디바이스 `SpeechRecognizer` + 규칙기반 파싱(키 불필요) — `ADR-004`. 클라우드/LLM은 후속(키 필요).
- **백엔드**: 메뉴·주문·매장·적립·알림은 Mock/Local 우선(`ADR-006`). 세션 토큰은 EncryptedDataStore(`SECURITY.md`).
- 보호 화면(주문 일부·MY·쿠폰·선물·주문내역)은 미인증(Guest/Expired) 시 재로그인 유도.
