# PRD: CafeMinsu (카페민수)

## 목표
동네 카페 "민수"의 단골이 **메뉴 주문·결제, 스탬프 적립·기프티콘, 음성 AI 주문**을
한 앱에서 빠르고 따뜻하게 처리하도록 한다. (네이티브 Android, 모바일 우선)

## 사용자
- **주 사용자**: 카페 민수를 반복 방문하는 단골 / 인근 직장인·주민 (모바일로 미리 주문하고 픽업)
- 1차 페르소나: "줄 서기 싫고, 적립은 챙기고 싶은 단골"
- (점주/매장 운영자는 MVP 대상 아님)

## 핵심 기능 (MVP)
1. **메뉴 탐색 + 주문 + 결제** — 카테고리별 메뉴 → 옵션 선택 → 장바구니 → 결제 → 주문 상태
2. **스탬프 적립 & 기프티콘** — 결제 시 스탬프 적립, 적립 현황 확인, 기프티콘 발급·보관·사용
3. **음성 AI 주문 (M-04)** — 음성으로 발화하면 STT→주문 파싱→장바구니에 자동 반영

## MVP 제외 사항
- **키오스크 모드** (매장 단말용) — 추후 별도 트랙
- 점주 어드민 / 매장 운영 콘솔
- 배달 / 외부 배달앱 연동
- 멤버십 등급·쿠폰 캠페인·추천인
- 다국어 (한국어 단일)

## 화면 인벤토리 (M-코드)
네비게이션·디자인 시스템(`DESIGN_SYSTEM.md` §9의 M-04 등)이 참조하는 화면 단일 목록.

| 코드   | 화면              | 핵심 내용                                       | MVP |
| ------ | ----------------- | ----------------------------------------------- | --- |
| M-01   | 홈 / 그리팅       | 인사, 최근/추천 메뉴, 진행 중 주문, 스탬프 요약 | ✓   |
| M-02   | 메뉴 목록         | 카테고리 탭, 메뉴 카드 그리드                   | ✓   |
| M-03   | 메뉴 상세 / 옵션  | 사이즈·온도·샷 등 옵션, 수량, 담기              | ✓   |
| M-04   | 음성 AI 주문      | 코랄 펄스 마이크, 실시간 transcript, 결과 확인  | ✓   |

> 음성 주문(M-04)은 홈/메뉴에서 진입하는 모달성 화면.

## 사용자 플로우 (핵심)
1. **표준 주문**: M-01 → M-02 → M-03(담기) → M-05 → M-06 → M-07 (스탬프 자동 적립)
2. **음성 주문**: M-01/M-02 → M-04(발화·확인) → M-05 → M-06 → M-07
3. **기프티콘 사용**: M-09 상세 → 매장 제시(바코드/QR)
4. **주요 예외**: 품절·영업마감·최소주문금액 미달 시 주문 차단/안내, 결제 실패·중복결제 방지, 세션 만료 시 재로그인 (처리 원칙은 `ARCHITECTURE.md` 에러/엣지 절, 보안은 `SECURITY.md`)

## 디자인
- 방향: **웜 크림 + 코랄 + 다크 네이비** 에디토리얼 (차분하고 손맛 있는 카페 톤)
- 상세 토큰·컴포넌트: `docs/DESIGN_SYSTEM.md` (단일 진실)
- 화면 적용·안티슬롭 원칙: `docs/UI_GUIDE.md`

## 가정 / 오픈 이슈 (구현 전 확정 필요)
- 결제: 실제 PG 연동 vs Mock 결제 (→ `ADR.md` ADR-006). API 키 필요 시 해당 step `blocked`.
- 음성 AI: STT(온디바이스 `SpeechRecognizer` vs 클라우드) + 주문 파싱 LLM 경로 (→ ADR-004).
- 백엔드: 메뉴·주문·적립 서버 존재 여부. MVP는 Mock/Local 우선 가정.
- 보안/세션: 토큰 저장(EncryptedDataStore)·HTTPS 강제·PG 토큰화 채택(→ `ADR-008~010`, `SECURITY.md`). 앱 내부 `AuthState`/세션 만료 처리는 `DATA_MODEL.md` 계약으로 고정하고, 로그인 제공자/UX(OTP/소셜 등)는 확정 필요.
