# 보안 가이드 (CafeMinsu)

모든 빌드 step에 주입되는 **보안 가드레일**. 기능 구현보다 본 요건을 우선한다.
요약 규칙은 `CLAUDE.md`, 에러/엣지 처리 절차는 `docs/ARCHITECTURE.md`를 함께 본다.

## 1. 인증 · 세션
- 인증 토큰·리프레시 토큰·세션은 **EncryptedDataStore**(또는 Keystore로 래핑한 저장소)에 보관. **평문(SharedPreferences/Room/파일) 저장 금지.**
- 로그아웃·세션 만료 시 토큰과 로컬 민감 데이터를 **즉시 와이프**.
- 토큰 만료(401)는 갱신 1회 시도 → 실패 시 재로그인. 토큰을 URL 쿼리·로그·화면에 노출하지 않는다.

## 2. 네트워크
- 모든 통신 **HTTPS 강제**. `android:usesCleartextTraffic="false"` + `networkSecurityConfig`로 평문 차단.
- 인증서 핀닝은 추후 옵션(과도한 고정으로 인한 운영 리스크 고려).
- 신뢰할 수 없는 입력은 서버 응답이라도 검증 후 사용.

## 3. 결제
- 카드 원번호(PAN)·CVC·유효기간을 **앱에 저장·로깅·캡처하지 않는다.** PG가 발급한 **토큰만** 사용.
- 결제 민감정보는 메모리 보유 최소화, 화면 캡처 방지 고려(`FLAG_SECURE`는 필요한 화면 한정).
- 중복 결제 방지는 **멱등키**로(상세 `ARCHITECTURE.md` 결제 안전 처리).

## 4. 개인정보 (PII)
- 전화번호·주문내역·적립정보는 **최소 수집·최소 보존**. 불필요한 영속화 금지.
- 로그·분석 이벤트에 PII를 남기지 않는다(마스킹: 전화번호 끝 4자리 등).
- 기프티콘 바코드/QR 등 민감 화면은 클립보드 복사·스크린샷 노출에 주의.

## 5. 로깅
- **릴리스 빌드에서 토큰·PII·결제정보 로깅 금지.** 디버그 로그는 릴리스에서 strip(R8/Timber 트리 분리 등).
- 예외 스택에 민감값이 실리지 않도록 메시지 정제.

## 6. 입력 · 딥링크 검증
- 수량·금액·옵션 등 사용자 입력은 **경계값 검증**(음수·과대 수량·미존재 옵션 거부).
- **딥링크/인텐트**는 스킴·호스트 **화이트리스트**로만 수락, 파라미터 위변조·미인증 경로 진입 차단.
- 외부 Intent로 받은 데이터는 신뢰하지 않고 검증·정규화.

## 7. 비밀키 · 빌드
- API 키·시크릿 **하드코딩·리포 커밋 금지.** `local.properties`/CI secret 주입, `.gitignore` 확인.
- 릴리스는 **R8/ProGuard 난독화 + 리소스 축소.** 디버그 전용 기능은 빌드 타입으로 분리.

## 8. 권한 최소화
- 마이크(RECORD_AUDIO)·알림(POST_NOTIFICATIONS) 등은 **런타임 요청 + rationale**. 사용 시점에만 요청.
- 불필요 권한 선언 금지. 권한 거부 시 대체 경로 제공(앱 크래시 금지).

## 9. BLE 근접(비콘) 스캔 — 제거됨
주문 스케줄링에 배선했던 **BLE 비콘 근접 신호는 제거되었다**(손님용 NFC 쿠폰 발급으로 대체). 비콘 스캐너
소스·DI·BLE 권한(`BLUETOOTH*`)은 더 이상 선언/사용하지 않는다. 순수 스케줄러(`domain/scheduling`)는
근접 입력 없이 동작한다.

## 10. 릴리스 전 체크리스트
- [ ] 토큰/세션이 EncryptedDataStore에 저장되고 로그아웃 시 와이프되는가
- [ ] cleartext 차단·HTTPS 강제가 매니페스트/설정에 반영됐는가
- [ ] 결제 정보 미저장·미로깅, 멱등키 적용됐는가
- [ ] 릴리스 로그에 토큰·PII·결제정보가 없는가
- [ ] 딥링크/인텐트 입력 검증이 있는가
- [ ] 시크릿이 코드/리포에 없고 난독화가 켜졌는가
