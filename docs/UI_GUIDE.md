# UI 적용 가이드 (CafeMinsu / Compose)

> 토큰·타이포·컴포넌트 **스펙의 단일 진실은 `DESIGN_SYSTEM.md`**.
> 본 문서는 그 토큰을 화면에 **어떻게 적용할지**와 **하지 말아야 할 것**(안티 AI-슬롭)을 다룬다.
> 토큰을 재정의하지 않는다. 색/치수는 항상 `DESIGN_SYSTEM.md` 토큰명으로 참조한다.

## 디자인 원칙
1. **카페의 손맛.** 마케팅 랜딩이 아니라 매일 쓰는 따뜻한 주문 도구. 웜 크림 `canvas` 베이스.
2. **차분한 강조.** 코랄 `primary`는 CTA·핵심 액션에만. 화면을 색으로 도배하지 않는다.
3. **콘텐츠 우선.** 메뉴·가격·상태가 주인공. 장식 요소는 최소화하고 여백으로 정돈한다.

## 안티 AI-슬롭 — 하지 마라
| 금지 사항 | 이유 |
|-----------|------|
| `MaterialTheme` 기본 보라/인디고 컬러 노출 | "AI = 보라색" 클리셰. 컬러 롤은 반드시 토큰으로 오버라이드 |
| 글래스모피즘(blur 반투명 카드) | AI 템플릿의 가장 흔한 징후 |
| 그라데이션 텍스트 / 네온 글로우 그림자 | AI SaaS 슬롭의 1번 특징 |
| "Powered by AI" 류 배지 | 기능이 아니라 장식. 사용자 가치 없음 |
| 모든 카드 동일한 과한 라운드(예: 24dp 일괄) | 카드 타입별 `radius-lg`/`radius-xl` 구분(§DESIGN_SYSTEM 5·7) |
| 배경 gradient orb / 블러 원형 장식 | 모든 AI 랜딩의 장식. 쓰지 않는다 |
| hex 리터럴 직접 입력 | 토큰만 사용. hex는 `Color.kt` 정의 한 곳뿐 |

## 색 적용 (요약 — 정의는 DESIGN_SYSTEM §2)
| 용도 | 토큰 |
|------|------|
| 화면 배경 | `canvas` |
| 카드/입력/리스트 셀 | `surface-card` |
| 다크 강조 카드(프로덕트) | `surface-dark` |
| 헤드라인/강조 텍스트 | `ink` |
| 본문 | `body` |
| 보조/placeholder | `muted` · 가장 약한 메타 `muted-soft` |
| CTA·활성·가격 강조 | `primary` |
| 구분선/비활성 보더 | `hairline` |
| 성공/경고/에러 | `success` / `warning` / `error` |

## 레이아웃
- 화면 사이드 패딩 `space-5`(20), 섹션 간격 `space-8`(32), 카드 내부 `space-4`~`space-5`.
- 좌측 정렬 기본. 본문·리스트 중앙 정렬 금지(타이틀/빈 상태 일러스트 예외).
- 단일 컬럼 모바일 폭. 메뉴 그리드는 2열까지.

## 타이포그래피 (정의는 DESIGN_SYSTEM §3)
| 용도 | 토큰 |
|------|------|
| 메인 그리팅 / 결제 완료 | `display` |
| 화면 타이틀 | `h1` |
| 섹션 헤더 | `h2` |
| 카드 타이틀 / 메뉴명 | `h3` |
| 본문 | `body` (강조 `bodyL`) |
| 가격 보조 / 메타 | `caption` / `meta` |

## 컴포넌트 사용 규칙
- 버튼/카드/칩/인풋/토스트는 `DESIGN_SYSTEM.md` §7 스펙대로 `ui/components/`에 만들고 **재사용**한다.
  화면에서 raw `Box`/`Button`에 색을 직접 칠하지 않는다.
- 메인 CTA는 `primary` 버튼 1개 원칙. 다크 카드 위 CTA는 `dark` variant.
- 가격·강조 수치는 `primary`, 일반 메타는 `muted`.

## 모션
- 허용: fade-in / slide-up(짧게, 200~300ms), 음성 펄스(§DESIGN_SYSTEM 9), 리스트 아이템 등장.
- 그 외 과한 스프링·무한 루프·패럴랙스 금지. 전환은 절제되고 빠르게.

## 아이콘
- 라인 아이콘, stroke 1.5px, 사이즈 20/24/32(§DESIGN_SYSTEM 8).
- 아이콘을 둥근 배경 박스로 감싸지 않는다(템플릿 느낌 방지). 색은 토큰만.

## 화면 상태 (필수 4종)
데이터를 부르는 모든 화면은 아래 4상태를 명시적으로 설계한다.
구현 계약은 `DATA_MODEL.md`의 `DataUiState`를 기준으로 하며, 화면별 sealed `UiState`를 만들 때도 의미를 유지한다.
- **Loading**: 콘텐츠 자리표시(스켈레톤/인디케이터). 전체 블로킹 스피너 남용 금지.
- **Empty**: 빈 상태 카피 + 다음 행동(예: "메뉴 보러가기"). 텍스트 `muted`.
- **Error**: 사용자 친화 한국어 메시지 + **재시도** 액션. 아이콘/보더 `error`.
- **Offline**: 오프라인 배너 + 캐시 데이터는 읽기 전용 노출.

## 돈 관련 액션 규칙
- 결제·주문 등 금전 액션은 **낙관적 UI 금지** — 서버 확정 후 성공 표시.
- 처리 중 버튼 비활성 + 중복 탭 차단(진행 인디케이터).
- 실패 시 무엇이 잘못됐고 다음에 뭘 할지 분명히(재시도 / 결제수단 변경 등).

## 점주(매장) 화면 적용
- 점주 화면은 고객과 **동일 토큰·컴포넌트**를 재사용한다. 점주 전용 색/토큰을 새로 만들지 않는다(배경 `canvas`, 카드/입력 `surface-card`, CTA·강조 수치 `primary`).
- 대시보드 통계 카드는 `surface-card`. 핵심 수치만 강조하고, 주의가 필요한 값(신규 대기 등)은 `primary`.
- 주문 큐 상태색: 신규 `warning` · 준비중 `primary` · 준비완료 `success`(점/태그로 표시). 상태 변경 버튼은 처리 중 비활성 — 운영 액션도 낙관적 UI 금지.
- 메뉴 품절 토글: 판매중은 `primary`(ON), 품절은 `hairline`(OFF) + 셀 디밍(`muted`) + `error` "품절" 태그.
- 매출 차트는 데이터 우선·단순 막대(`accent-soft`, 당일 `primary`). 그라데이션/글로우/3D·도넛 장식 금지(§안티 AI-슬롭).
- 점주 데이터 화면도 **4상태(Loading/Empty/Error/Offline)**를 유지한다(예: "새 주문이 없어요" Empty).

## 접근성
- 터치 타깃 최소 48dp. 텍스트 대비는 토큰 조합(`ink`/`body` on `canvas`/`surface-card`)으로 확보.
- 음성 주문은 시각 transcript를 항상 병행 노출.
